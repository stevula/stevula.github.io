Regular Expressions

30 October, 2015

Regular expressions (AKA Regexes or RegExps) are a powerful syntax for matching string patterns in programming languages and text editors. Because they are so incredibly useful and widely used, it is worth any new programmer's time to learn what they can do, even if you don't have the time to memorize the syntax just yet.

If you're reading this, chances are you're in my DBC cohort. Remember when we had to make a list of students in our class for that solo challenge in Ruby? As you may recall, there was a list of students in the Canvas "People" tab. It looks nice and tabular in Canvas, but it didn't exactly provide a neatly formatted array of names for your Ruby code:

When I copy/paste that into Sublime Text, it looks like this:

So, we've got a couple problems here:

  1. The student names appear twice (and not always with the same spelling)
  2. The cohort name appears on every third line, which is useless to us
  3. The user-type "Student" appears on every fourth line, which is also useless to us

To get a list of names, you might say we just want to capture every fourth line, starting with line 1. Luckily, we can use a RegEx to do this via Sublime's "Find" and "Replace" functions.

  1. Enter .*\n in the "Find" field to select each line individually
    • . matches any single character:
    • + operator modifies the preceding RegEx (.) to match all consecutive characters, instead of just single ones.
    • \n matches newline characters (the end of each line).
    • Together, these create an expression that matches a series of all characters from the start to the end of the line:
  2. With a slight modification, we can make it grab each group of four lines: (.*\n){4}
    • Enclose the expression from step 1 in parentheses to make sure the line-selector gets evaluated first: (.*\n)
    • Add {4} to select every four of that pattern:
  3. You now (conveniently) have a cursor on every fourth line, which you can use to copy/paste the name of the students or easily delete the unnecessary lines:

You can also use RegExes to "comma-ify" numbers. Let's say we have the number 10000 and we want to make it look like 10,000. It's pretty easy if the number only needs one comma:

  1. Enter \d in the "Find" field to select each digit:
  2. Add a {3} to modify the previous RegExp to match groups of three digits: \d{3}:
  3. Enclose the previous RegExp in parentheses and add a $ to match from the end instead of the beginning of the line: (\d{3})$
  4. In the "Replace" field, enter ,$1.
    • The , is just a regular comma character (",").
    • The $1 captures the matched string from the "Find" field ("000").
    • Together, these replace the original string with a comma + original string:

Unfortunately, it's a bit trickier to add commas to a larger number, like 1000000. The RegEx above only matches the first set of three digits from the end, but this would require it to match every set of three digits.

If you're interested in figuring out how to do that, there is a fantastic tutorial on RegExes on the Bastard's Book of Ruby website. Once you're familiar with how they work, you might also appreciate this RegEx cheat-sheet made by the same author. For a more comprehensive reference on RegExes, check out Regular-Expressions.info.
