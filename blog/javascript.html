JavaScript vs Ruby

21 October, 2015

One of the key differences between JavaScript and Ruby is how variables are scoped.

In Ruby, we have several different levels of variable scope:

foobar => a local variable @foobar => an instance variable @@foobar => a class variable $foobar => a global variable FOOBAR => a constant (also global)

In JavaScript, there are only two levels of variable scope:

var foobar => a local variable (definition) foobar => a global variable (definition) => note: both are referenced the same way by typing "foobar", but they must be defined with the different syntaxes above

Ruby and JavaScript also mean something different when they consider a variable "local". In Ruby, a local variable can only be accessed from within the method, loop, or block in which it was declared.

[1,2,3].each do |x|   print x end => "123" print x => undefined local variable or method 'x'

x cannot be accessed from outside the block in the example above.

In JavaScript, local variables only exist inside of functions. Local variables can be accessed from within the function in which they were defined, including from any functions nested inside of that function. As in Ruby, arguments are also treated as local variables in the function/method to which they are passed.

function printInConsole(string) {   console.log(string); } printInConsole("Hello, world"); => "Hello, world!" console.log(string); => ReferenceError: string is not defined

string cannot be accessed from outside the function in the example above.

As you can see, JavaScript bases scope on functions, whereas Ruby scope is based mainly on objects and methods. That shouldn't be surprising, since Ruby was built from the ground up as an extremely object-oriented language. In some ways the concept of scope is much simpler in JavaScript, but don't let that fool you. Scope can get pretty complicated once you consider closures and hoisting - two special features of JavaScript that pertain to variables and scope.
